# T-05 — 테스트 재작성 우선순서를 어디서 시작할지

## 문맥
- 첫 재작성 타깃은 결함 탐지 패턴을 결정한다. 먼저 잡는 영역이 나머지 테스트 스타일의 템플릿이 된다.

## 선택지별 트레이드오프
### A) `session/text/callback` 먼저
- 트레이드오프: 사용자 체감 경로를 빨리 보호하지만 도메인 순수성 규칙이 늦게 정리될 수 있다.
- 되돌리기 비용(코드 변경량): 300줄 미만

### B) domain pure 먼저
- 트레이드오프: 기초 체력이 강해지지만 실제 사용자 플로우 회귀를 늦게 잡는다.
- 되돌리기 비용(코드 변경량): 300줄 미만

### C) e2e 먼저
- 트레이드오프: 시스템 관점 가시성은 좋지만 실패 원인 국소화가 어렵다.
- 되돌리기 비용(코드 변경량): 100줄 미만

## 질문 중요도
- 최대 되돌리기 비용: 300줄 미만
- 중요도: 높음 (되돌릴 때 바뀌는 코드 경로의 폭 기준)

## 결정 체크포인트
- 선택 즉시 영향받는 영역: 포트 인터페이스, 핸들러/서비스 분기, 테스트 전략
- 결정 후 확인할 지표: 회귀 실패율, PR 크기, 리팩토링 단계 충돌 건수
