# T-08 — 회귀 기준을 golden 중심으로 갈지 unit 중심으로 갈지

## 문맥
- 회귀 기준은 버그를 "어떤 형태로" 막을지에 대한 선언이다. 기준이 흐리면 테스트 양은 많아도 방어력은 약해진다.

## 선택지별 트레이드오프
### A) 핵심 시나리오 golden 테스트 필수
- 트레이드오프: 행동 보존성은 높지만 결과 스냅샷 관리 비용이 증가한다.
- 되돌리기 비용(코드 변경량): 300줄 미만

### B) unit 중심 대체
- 트레이드오프: 원인 추적이 빠르지만 엔드투엔드 출력 불일치를 늦게 발견할 수 있다.
- 되돌리기 비용(코드 변경량): 300줄 미만

## 질문 중요도
- 최대 되돌리기 비용: 300줄 미만
- 중요도: 높음 (되돌릴 때 바뀌는 코드 경로의 폭 기준)

## 결정 체크포인트
- 선택 즉시 영향받는 영역: 포트 인터페이스, 핸들러/서비스 분기, 테스트 전략
- 결정 후 확인할 지표: 회귀 실패율, PR 크기, 리팩토링 단계 충돌 건수
