# W-03 — 멀티유저 스레드에서 interrupt 권한 모델을 어떻게 둘지

## 문맥
- 현재 soma-work는 owner/currentInitiator 개념이 있고, 잘못 설계하면 타인 세션 중단/경합 문제가 바로 발생한다.

## 선택지별 트레이드오프
### A) owner-only
- 트레이드오프: 안전하지만 협업 중 즉시 개입성이 떨어진다.
- 되돌리기 비용(코드 변경량): 100줄 미만

### B) initiator 우선 + owner override
- 트레이드오프: 현실적 절충안이지만 상태 전이 규칙을 명확히 해야 한다.
- 되돌리기 비용(코드 변경량): 300줄 미만

### C) role-based(관리자/참여자)
- 트레이드오프: 정책 유연성은 높지만 권한 동기화/운영 복잡도가 커진다.
- 되돌리기 비용(코드 변경량): 500줄 미만

## 질문 중요도
- 최대 되돌리기 비용: 500줄 미만
- 중요도: 매우 높음 (되돌릴 때 바뀌는 코드 경로의 폭 기준)

## 결정 체크포인트
- 선택 즉시 영향받는 영역: 식별자 스키마, 세션 키, 권한/저장소 partition, 채널 어댑터 계약
- 결정 후 확인할 지표: 세션 충돌 건수, 권한 오판률, adapter 변경량, cross-repo 드리프트 건수
