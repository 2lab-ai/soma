# M-02 — 1차 타겟을 session/text/provider-port 중 어디로 둘지

## 문맥
- 첫 타겟은 위험을 어디서 먼저 제거할지 결정한다. 우선순위가 곧 초기 학습 곡선이다.

## 선택지별 트레이드오프
### A) `session.ts` 분해 먼저
- 트레이드오프: 핵심 복잡도를 먼저 줄이지만 초기 충돌 범위가 넓다.
- 되돌리기 비용(코드 변경량): 300줄 미만

### B) `text.ts` 분해 먼저
- 트레이드오프: 사용자 체감 경로를 빠르게 개선하지만 근본 provider 결합은 남는다.
- 되돌리기 비용(코드 변경량): 300줄 미만

### C) provider port 먼저
- 트레이드오프: 확장 기반은 빨리 잡지만 현재 핸들러 문제 해결은 늦어진다.
- 되돌리기 비용(코드 변경량): 300줄 미만

## 질문 중요도
- 최대 되돌리기 비용: 300줄 미만
- 중요도: 높음 (되돌릴 때 바뀌는 코드 경로의 폭 기준)

## 결정 체크포인트
- 선택 즉시 영향받는 영역: 포트 인터페이스, 핸들러/서비스 분기, 테스트 전략
- 결정 후 확인할 지표: 회귀 실패율, PR 크기, 리팩토링 단계 충돌 건수
