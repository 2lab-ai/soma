# W-05 — v3에서 Slack 최소 지원 범위를 어디까지 둘지

## 문맥
- 요구사항은 “지금 합치지 않되 최소 지원을 고려”하는 것이다. 범위를 명확히 해야 일정과 리스크를 통제할 수 있다.

## 선택지별 트레이드오프
### A) text/thread만
- 트레이드오프: 가장 안전한 최소 범위지만 실제 사용성은 제한된다.
- 되돌리기 비용(코드 변경량): 100줄 미만

### B) +file/reaction
- 트레이드오프: 실사용에 가까워지지만 어댑터 표면적이 커진다.
- 되돌리기 비용(코드 변경량): 300줄 미만

### C) +interactive(choice/form)
- 트레이드오프: UI parity에 가깝지만 v3 초기 범위를 과도하게 확장할 수 있다.
- 되돌리기 비용(코드 변경량): 500줄 미만

## 질문 중요도
- 최대 되돌리기 비용: 500줄 미만
- 중요도: 매우 높음 (되돌릴 때 바뀌는 코드 경로의 폭 기준)

## 결정 체크포인트
- 선택 즉시 영향받는 영역: 식별자 스키마, 세션 키, 권한/저장소 partition, 채널 어댑터 계약
- 결정 후 확인할 지표: 세션 충돌 건수, 권한 오판률, adapter 변경량, cross-repo 드리프트 건수
