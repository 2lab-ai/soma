# M-03 — PR 크기 제한을 얼마나 강하게 둘지

## 문맥
- PR 크기 제한은 리뷰 품질과 개발 속도의 균형장치다.

## 선택지별 트레이드오프
### A) 500 LOC 이하
- 트레이드오프: 리뷰 품질은 높지만 PR 수가 늘어난다.
- 되돌리기 비용(코드 변경량): 10줄 미만

### B) 1000 LOC 이하
- 트레이드오프: 속도와 품질의 절충안이지만 일부 큰 변경은 여전히 부담된다.
- 되돌리기 비용(코드 변경량): 10줄 미만

### C) 제한 없음
- 트레이드오프: 속도는 빠를 수 있으나 결함 누수와 리뷰 피로가 크게 증가한다.
- 되돌리기 비용(코드 변경량): 100줄 미만

## 질문 중요도
- 최대 되돌리기 비용: 100줄 미만
- 중요도: 보통 (되돌릴 때 바뀌는 코드 경로의 폭 기준)

## 결정 체크포인트
- 선택 즉시 영향받는 영역: 포트 인터페이스, 핸들러/서비스 분기, 테스트 전략
- 결정 후 확인할 지표: 회귀 실패율, PR 크기, 리팩토링 단계 충돌 건수
