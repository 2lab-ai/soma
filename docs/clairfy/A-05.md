# A-05 — interrupt 우선순위를 절대 최우선으로 둘지 정책형으로 둘지

## 문맥
- interrupt 정책은 사용자 체감 응답성과 내부 상태 안정성의 교환점이다.

## 선택지별 트레이드오프
### A) 절대 최우선(큐 우회)
- 트레이드오프: 체감 반응성은 높지만 도중 상태 정리 누락 가능성이 있다.
- 되돌리기 비용(코드 변경량): 100줄 미만

### B) 상태 기반 정책 우선
- 트레이드오프: 안정성은 높지만 사용자 입장에서는 중단이 늦게 느껴질 수 있다.
- 되돌리기 비용(코드 변경량): 100줄 미만

## 질문 중요도
- 최대 되돌리기 비용: 100줄 미만
- 중요도: 보통 (되돌릴 때 바뀌는 코드 경로의 폭 기준)

## 결정 체크포인트
- 선택 즉시 영향받는 영역: 포트 인터페이스, 핸들러/서비스 분기, 테스트 전략
- 결정 후 확인할 지표: 회귀 실패율, PR 크기, 리팩토링 단계 충돌 건수
