# A-07 — reaction/keyboard를 Abstraction A 계약에 포함할지

## 문맥
- UI 보조 기능을 포트에 올리면 채널 독립성이 올라가지만 인터페이스 비대화가 발생할 수 있다.

## 선택지별 트레이드오프
### A) Port에 포함
- 트레이드오프: 확장 시 일관성은 높지만 core가 UI 관심사에 더 노출된다.
- 되돌리기 비용(코드 변경량): 300줄 미만

### B) Telegram adapter 내부 private
- 트레이드오프: core는 깔끔하지만 채널별 기능 편차를 흡수하기 어렵다.
- 되돌리기 비용(코드 변경량): 300줄 미만

## 질문 중요도
- 최대 되돌리기 비용: 300줄 미만
- 중요도: 높음 (되돌릴 때 바뀌는 코드 경로의 폭 기준)

## 결정 체크포인트
- 선택 즉시 영향받는 영역: 포트 인터페이스, 핸들러/서비스 분기, 테스트 전략
- 결정 후 확인할 지표: 회귀 실패율, PR 크기, 리팩토링 단계 충돌 건수
