# W-06 — Slack 출력 계약을 text 중심으로 둘지 status/interactive까지 포함할지

## 문맥
- 출력 계약은 채널 어댑터 설계의 핵심이다. 너무 얇으면 기능 누락, 너무 두꺼우면 core가 UI 관심사에 잠긴다.

## 선택지별 트레이드오프
### A) text only
- 트레이드오프: 간단하지만 상태 표현/피드백 경험이 약하다.
- 되돌리기 비용(코드 변경량): 300줄 미만

### B) +status/reaction
- 트레이드오프: 운영 가시성을 확보하면서도 복잡도는 통제 가능하다.
- 되돌리기 비용(코드 변경량): 100줄 미만

### C) +interactive output
- 트레이드오프: 풍부한 UX 가능하지만 포트 계약이 무거워진다.
- 되돌리기 비용(코드 변경량): 300줄 미만

## 질문 중요도
- 최대 되돌리기 비용: 300줄 미만
- 중요도: 높음 (되돌릴 때 바뀌는 코드 경로의 폭 기준)

## 결정 체크포인트
- 선택 즉시 영향받는 영역: 식별자 스키마, 세션 키, 권한/저장소 partition, 채널 어댑터 계약
- 결정 후 확인할 지표: 세션 충돌 건수, 권한 오판률, adapter 변경량, cross-repo 드리프트 건수
